## Описание

Check-List - это многофункциональное приложение для управления задачами, которое предоставляет два способа взаимодействия:

1. **Командная строка (CLI)** - для локального управления задачами
2. **Telegram бот** - для управления задачами через мессенджер Telegram

Приложение позволяет создавать, просматривать, редактировать, отмечать выполненными и удалять задачи. Все данные сохраняются в формате JSON, что обеспечивает простоту хранения и переносимость.

## Возможности

### CLI утилита (todo)

- Добавление новых задач
- Просмотр всех задач
- Просмотр отдельно выполненных и невыполненных задач
- Редактирование текста задач
- Отметка задач как выполненных/невыполненных
- Удаление задач
- Очистка всего списка задач
- Настройка пути хранения файлов задач
- Настройка имени файла для списка задач

### Telegram бот (tg-bot)

- Все основные функции CLI утилиты доступны через Telegram
- Удобное взаимодействие через команды бота
- Возможность управления задачами из любого места через мессенджер

## Требования для сборки

### Linux (Ubuntu/Debian)

Для сборки проекта необходимы следующие зависимости:

- **Компилятор C++** с поддержкой C++23 (GCC 13 или выше)
- **CMake** версии 3.31 или выше
- **Conan** версии 2.22.2 или выше
- **Python 3** (для работы Conan)
- **Git** (для клонирования репозитория)

### Зависимости проекта

Проект использует следующие библиотеки:

- **Boost** 1.89.0 (библиотеки system, beast)
- **OpenSSL** 3.6.0
- **nlohmann_json** 3.12.0
- **magic_enum** 0.9.7

Все зависимости автоматически устанавливаются через Conan при сборке проекта.

## Инструкция по сборке на Linux

### 1. Клонирование репозитория

```bash
git clone https://github.com/Joraslav/check-list.git
cd check-list
```

### 2. Настройка окружения

Создайте окружение для сборки (Debug или Release):

```bash
chmod +x scripts/create_env.sh
./scripts/create_env.sh Release
```

или для отладочной сборки:

```bash
./scripts/create_env.sh Debug
```

### 3. Сборка проекта

Используйте скрипт сборки:

```bash
chmod +x scripts/build.sh
./scripts/build.sh
```

Или выполните сборку вручную:

```bash
cmake --preset conan-release -DBUILD_TEST=ON -DDEV_MODE=OFF
cmake --build --preset conan-release
```

### 4. Запуск тестов

После сборки можно запустить тесты:

```bash
ctest --test-dir build/Release
```

## Использование

### CLI утилита

После сборки исполняемый файл `todo` будет находиться в директории `build/Release/`:

```bash
# Просмотр справки
./todo help

# Добавление задачи
./todo add "Покупка продуктов"

# Просмотр всех задач
./todo list

# Просмотр невыполненных задач
./todo list pending

# Просмотр выполненных задач
./todo list completed

# Отметить задачу как выполненную/невыполненную
./todo done 0

# Редактировать задачу
./todo edit 0 "Покупка овощей и фруктов"

# Удалить задачу
./todo remove 0

# Очистить все задачи
./todo clear

# Настройка пути хранения файлов
./todo config path "/home/user/my_tasks"

# Настройка имени файла
./todo config name "my_todo_list.json"
```

### Telegram бот

Исполняемый файл `tg-bot` также будет находиться в директории `build/Release/`:

```bash
./tg-bot
```

**Важно:** Для работы бота необходимо указать действительный токен Telegram бота в коде (в файле `src/bot.cpp`).
